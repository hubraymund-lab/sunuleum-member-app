# 개발일지 - 멀티 호점(브랜치) 시스템 도입

## 날짜: 2026-03-28

## 프로젝트 개요
- **프로젝트명**: 수눌음 통합 회원 관리 시스템
- **기술 스택**: React + Vite + Tailwind CSS v4 + React Router + Supabase + TanStack Query + Lucide React
- **배포**: Vercel (https://sunuleum-member-app.vercel.app)

## 목표
- 현재 단일 시스템을 멀티 호점(브랜치) 구조로 전환
- 첫 번째 호점: 육아 나눔터 6호점 (시범 운영)
- 각 호점별 관리자 지정 기능 구현
- 향후 다른 호점 추가 가능한 확장 가능한 구조 설계

## 설계 결정사항
- 김님 = super_admin (모든 호점 관리 가능)
- 회원은 여러 호점에 동시 소속 가능
- URL 구조: `/branch/:branchId/...` (호점별 라우팅)
- profiles.system_role = 'super_admin' | 'user' (시스템 전체)
- branch_members.role = 'admin' | 'member' (호점별)

## 작업 내용

### Phase 1: DB 스키마 (완료)
- `branches` 테이블 신설 (호점 정보)
- `branch_members` 테이블 신설 (회원-호점 매핑 + 역할)
- `profiles`에 `system_role` 컬럼 추가
- 기존 8개 테이블에 `branch_id` 컬럼 추가
- 파일럿 호점 "육아 나눔터 6호점" 삽입 + 기존 데이터 백필
- RLS 정책 전면 개편 (호점별 관리자 체크)
- 헬퍼 함수: `is_super_admin()`, `is_branch_admin()`, `is_branch_member()`
- 마이그레이션 파일: `supabase/migration-multi-branch.sql`

### Phase 2: 프론트엔드 인프라 (완료)
- `src/lib/branch.jsx` - BranchContext (호점 선택/전환)
- `src/lib/auth.jsx` - isSuperAdmin 추가
- `src/components/guards/BranchGuard.jsx` - 호점 접근 권한 체크
- `src/components/guards/BranchAdminGuard.jsx` - 호점 관리자 체크
- `src/components/guards/SuperAdminGuard.jsx` - 슈퍼관리자 체크
- `src/pages/BranchSelect.jsx` - 호점 선택 페이지
- `src/components/RootRedirect.jsx` - / → 호점 자동 리다이렉트

### Phase 3: 레이아웃 & 라우팅 (완료)
- `App.jsx` 라우팅 전면 재구성 (`/branch/:branchId/` 기반)
- `MemberLayout.jsx` - 호점 이름 표시, 호점 전환 드롭다운
- `AdminLayout.jsx` - 호점별 관리자 네비게이션
- `src/components/layout/SuperAdminLayout.jsx` 신규 생성

### Phase 4: 기존 페이지 업데이트 (완료)
- 회원 페이지 7개: branch_id 필터 추가 (Home, Programs, Toys, MyEnrollments, MyAttendance, MyFees, MyToyRentals)
- 관리자 페이지 9개: branch_id 필터 + branch_members 기반 조회로 전환
- Members.jsx 대폭 리팩토링: profiles → branch_members 조인 구조

### Phase 5: 슈퍼관리자 페이지 (완료)
- `src/pages/super-admin/Dashboard.jsx` - 전체 호점 대시보드
- `src/pages/super-admin/Branches.jsx` - 호점 CRUD + 회원/관리자 관리
